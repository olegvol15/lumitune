# Frontend Onboarding

## Stack

- React 19 + TypeScript
- Vite 7
- Tailwind CSS 3
- TanStack Router file-based routes
- TanStack Query
- Zustand
- Axios
- Framer Motion
- Recharts
- ESLint 9 + `typescript-eslint` + React Hooks rules
- Icons from `lucide-react` and `react-icons`

## Project Structure

```txt
src/
  api/                    # axios transport functions by backend resource
  assets/
    media/                 # local images used in UI
  components/
    admin/                 # admin panel UI and modals
    auth/                  # auth-only UI pieces
    home/                  # home page sections
    layout/                # app shell, nav, sidebars, players
    notifications/         # notification components
    player/                # audio engine and playback controls
    profile/               # profile page sections and modals
    settings/              # settings controls
    ui/                    # reusable shared UI primitives
  hooks/                  # React hooks around API and state flows
  lib/                    # configured clients, query client, i18n, motion helpers
  routes/                 # TanStack Router file routes
    admin/                 # admin routes
    auth/                  # user auth routes
  store/                  # Zustand stores
  types/                  # shared TypeScript models and API types
  utils/                  # shared helper and mapper functions
  index.css               # global styles and Tailwind layers
  main.tsx                # app bootstrap
  routeTree.gen.ts        # generated by TanStack Router plugin
```

## Routing

- Add pages under `src/routes`.
- Use `createFileRoute('/path')({ component: Page })`.
- Dynamic routes use file names like `album.$id.tsx`.
- User auth routes live under `src/routes/auth`.
- Admin routes live under `src/routes/admin`.
- Use TanStack Router navigation APIs instead of raw anchors for in-app navigation.
- Do not edit `src/routeTree.gen.ts`; it is generated by the router plugin.

## API, Hooks, and State

- API modules in `src/api` contain request functions only.
- Hooks in `src/hooks` compose API calls, query behavior, and UI-facing mutations.
- Zustand stores in `src/store` hold global client state such as auth, admin auth, player, playlists, language, and theme.
- `src/lib/apiClient.ts` is the authenticated Axios client. It attaches user or admin bearer tokens and attempts refresh on eligible `401` responses.
- `src/lib/publicApiClient.ts` is for unauthenticated requests.
- Keep backend-to-UI mapping logic in `src/utils` instead of embedding it in components.

## Components

- Reusable primitives belong in `src/components/ui`.
- Route-specific UI belongs in the nearest domain folder.
- Admin CRUD modal and table support belongs in `src/components/admin`.
- Layout components should stay in `src/components/layout`.
- Keep components focused. Move repeated JSX or formatting rules into local helpers or shared components once reuse is clear.

## Player Flow

- `src/components/player/AudioEngine.tsx` owns the single global audio element.
- UI controls call actions in `playerStore`.
- Streams are loaded from `/api/songs/:id/stream`.
- Progress, duration, play state, volume, shuffle, and repeat are synchronized through the store.

## Styling

- Use Tailwind utilities first.
- Shared global styles and component classes live in `src/index.css`.
- Prefer existing UI primitives and utility patterns before adding new ones.
- Keep interactive controls accessible with visible focus states and disabled states.

## Media

- Put app media in `src/assets/media`.
- Prefer local optimized assets over random remote URLs.
- Use the shared cover/fallback components when rendering catalog artwork.
- Do not commit large generated media unless it is needed by the UI.

## Commands

From the repository root:

```bash
pnpm dev:frontend
pnpm lint:frontend
pnpm typecheck:frontend
pnpm build:frontend
```

From `frontend/`:

```bash
pnpm dev
pnpm lint
pnpm typecheck
pnpm build
pnpm preview
```

## Delivery Checklist

1. `pnpm lint:frontend` passes for frontend changes.
2. `pnpm typecheck:frontend` passes for TypeScript changes.
3. `pnpm build:frontend` passes before handing off production-facing UI changes.
4. New routes are added through `src/routes`.
5. `src/routeTree.gen.ts` is not edited manually.
6. New API calls are placed in `src/api` and consumed through hooks where practical.
7. New shared types are placed in `src/types`, not duplicated inside stores or components.
