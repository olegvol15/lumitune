---
name: refactor-propose
description: Proposes structural improvements to code without changing its external behavior.
---

When proposing a refactor:

1. Identify areas of high complexity or "code smells"
2. Present the proposal as follows:

## Current State
Briefly describe the current pain point (e.g., "Deeply nested conditionals").

## Proposed Change
Show the refactored code snippet.

## Benefits
- How this improves maintainability
- How this reduces cognitive load
- Impact on testability
